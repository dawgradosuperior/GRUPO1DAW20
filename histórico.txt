- Nos descargamos el proyecto con el que vamos a trabajar en nuestros equipos y cada uno borramos los archivos
que no vamos a necesitar para no tener problemas al subirlos a GitHub.
- Joaquín crea el repositorio en GitHub sobre el que vamos a trabajar, creando la rama "master", que es sobre
la que vamos a trabajar, y da acceso al resto de integrantes del grupo.
- Ángela crea la base de datos que vamos a necesitar para que el proyecto funcione correctamente en su hosting de CDmon.
- Cada uno sube su parte correspondiente del proyecto en GitHub desde su equipo usando Git con sus comandos 
correspondientes (subiendo la última parte, se fusionan por error 2 ramas, aunque no parece provocar ningún problema).
Las carpetas vacías no se suben, ya que GitHub trabaja con archivos y no las acepta.

usando SSH y de nuevo los comandos Git anteriormente usados, esta vez para pasar el proyecto de GitHub a CDmon.
- Para tener una mayor seguridad, decidimos subir el archivo Config.php sin modificar a GitHub, y lo modificamos
en CDmon con la herramienta de acceso web de FTP de CDmon, introduciendo los datos de la base de datos y el usuario
usado para acceder a ella.
- Una vez subido el proyecto, todos revisamos que funcione correctamente.
